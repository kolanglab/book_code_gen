# 用語集

本書で登場した主な用語を、初学者向けに簡潔にまとめます。詳しい説明は各章の本文を
参照してください。

## コンパイラの構造に関する用語

**コンパイラ（compiler）**
: ある言語のプログラムを、等価な別の言語（多くは機械語やバイトコード）のプログラムへ
  変換するソフトウェア。前半をフロントエンド、後半をバックエンドと呼ぶ。

**フロントエンド / バックエンド（front end / back end）**
: フロントエンドは字句解析・構文解析・意味解析など、入力言語を解析して中間表現を
  作るまでの部分。バックエンドは中間表現から最適化とコード生成を行う部分。本書は
  おもにバックエンドを扱う。

**抽象構文木（Abstract Syntax Tree, AST）**
: プログラムの構造を木で表したデータ。構文上は必要だが意味に影響しない要素（括弧
  など）を省き、計算の本質だけを残す。本書のコード生成の出発点。

**中間表現（Intermediate Representation, IR）**
: フロントエンドとバックエンドの橋渡しをする、言語にも機械にも依存しない内部表現。
  木構造・三番地コード・スタックベースなど複数の形式がある。最適化を行う場所でもある。

**コード生成（code generation）**
: 中間表現（や AST）から、実際に動く命令列（バイトコードや機械語）を作り出す処理。
  本書の中心テーマ。

## 命令と機械に関する用語

**仮想マシン（Virtual Machine, VM）**
: ソフトウェアとして実現された架空の計算機。物理 CPU に依存しないバイトコードを実行
  する。Java の JVM や Ruby の YARV が代表例。

**バイトコード（bytecode）**
: 仮想マシンが実行する命令列。特定の CPU に依存しないので移植しやすい。

**スタックマシン / レジスタマシン（stack machine / register machine）**
: スタックマシンは計算用の値をスタックに積み下ろしして処理する方式。レジスタマシンは
  名前付きの少数の高速な置き場（レジスタ）を使う方式で、現実の CPU はこちら。

**レジスタ（register）**
: CPU 内部にある、ごく少数で非常に高速な値の置き場。数が有限であることがレジスタ
  割り付けの問題を生む。

**仮想レジスタ（virtual register）**
: 命令選択の段階で、個数を気にせず使う仮想的なレジスタ。あとでレジスタ割り付けにより
  有限の物理レジスタへ割り当て直される。

**プログラムカウンタ（program counter, pc）**
: いま実行している命令の位置を表す値。ジャンプ命令はこれを書き換えて実行の流れを
  曲げる。

## コード生成の技法に関する用語

**後行順走査（post-order traversal）**
: 木をたどるとき、子をすべて処理してから自分を処理する順番。スタックマシン向けの
  コード生成の基本となる走査順。

**三番地コード（three-address code）**
: ひとつの命令が結果1つと入力2つの「番地」を持つ線形の中間表現。複雑な式を一度に
  ひとつの演算へ分解した形。

**ジャンプ / ラベル（jump / label）**
: ジャンプは実行を別の位置へ移す命令。ラベルはその移り先を指す命令列中の目印。
  `if` や `while` などの制御構造の実現に使う。

**バックパッチ（backpatching）**
: ジャンプ先がまだ決まっていない段階で命令を出しておき、後から実際の位置を埋める
  手法。

**短絡評価（short-circuit evaluation）**
: `a && b` で `a` が偽なら `b` を評価しないように、論理演算で必要な側だけを評価
  すること。論理演算は分岐としてコード生成する必要がある。

**基本ブロック / 制御フローグラフ（basic block / control flow graph, CFG）**
: 基本ブロックは途中に分岐の出入りがないひとかたまりの命令列。CFG はそれらを節点とし、
  実行が進みうる関係を辺で結んだグラフ。多くの解析・最適化の土台。

**呼び出し規約（calling convention）**
: 関数の呼び出し元と呼び出し先のあいだで、引数の渡し方・戻り値の返し方・フレームの
  片付け方などをあらかじめ取り決めたもの。

**スタックフレーム / 呼び出しスタック（stack frame / call stack）**
: スタックフレームは関数呼び出しごとに作られる作業領域（ローカル変数や戻り先を持つ）。
  呼び出しスタックはそれらを積み下ろしして管理する仕組み。

## 命令選択・レジスタ割り付けに関する用語

**命令選択（instruction selection）**
: 同じ計算を実現できる複数の機械語命令の組み合わせから、良いものを選ぶ処理。木の
  タイル張りとして定式化できる。

**最大限の食いつき（maximal munch）**
: 命令選択で、当てはまる最大のタイル（命令）を貪欲に選ぶ方針。単純で高速。

**レジスタ割り付け（register allocation）**
: 仮想レジスタを有限の物理レジスタへ割り当てる処理。グラフ彩色として定式化される。

**生存区間 / 生存解析（live range / liveness analysis）**
: 生存区間は、変数の値が作られてから最後に使われるまでの範囲。生存解析は CFG 上で
  どの変数がどこで生きているかを求める解析。

**干渉グラフ（interference graph）**
: 同時に生きている（同じレジスタにできない）変数どうしを辺で結んだグラフ。レジスタ
  割り付けはこのグラフの彩色問題になる。

**スピル（spill）**
: レジスタが足りないとき、値を一時的にメモリ（スタックフレーム）へ退避すること。
  メモリアクセスは遅いので、最小限にしたい。

## 最適化・実行時コンパイルに関する用語

**最適化（optimization）**
: プログラムの意味を変えずに、コードを速くまたは小さくする変換。

**のぞき穴最適化（peephole optimization）**
: 命令列を小さな窓で眺め、その範囲の無駄なパターンを等価でより短いパターンに書き換える
  最適化。

**定数畳み込み / 定数伝播（constant folding / constant propagation）**
: 定数畳み込みはコンパイル時に計算できる演算を計算してしまう最適化。定数伝播は定数と
  分かった変数の使用箇所を定数に置き換える最適化。

**共通部分式の除去 / 不要コードの除去（CSE / DCE）**
: CSE は同じ計算の重複を1回にまとめる最適化。DCE は結果が使われない計算を削除する
  最適化。

**静的単一代入形式（Static Single Assignment, SSA）**
: すべての変数がちょうど一度だけ代入されるよう名前を振った中間表現。値の定義位置が
  一意に分かり、多くの最適化が簡単になる。合流点では φ関数を使う。

**JIT コンパイル（Just-In-Time compilation）**
: プログラムの実行中に、よく実行される部分を機械語へコンパイルする技法。実行時情報を
  使った最適化ができる。

**脱最適化（deoptimization）**
: JIT が置いた楽観的な仮定が実行時に崩れたとき、安全なインタプリタ実行へ引き返す
  仕組み。
